# AlgoSort ⚙️

> **コードを並び替えて、アルゴリズムをマスターしよう**

競技プログラミングのコードを正しい順序に並び替えるパズルゲームです。  
タップ・ドラッグ操作でコードブロックを並べ、アルゴリズムの構造を楽しく学べます。

🔗 **プレイはこちら**: [https://algo-sort.vercel.app](https://algo-sort.vercel.app)

---

## 🎮 ゲームの遊び方

1. **カテゴリ / 難易度を選ぶ**  
   「ランダムに挑戦」または「カテゴリから選ぶ」でステージを開始します。

2. **コードブロックを並び替える**  
   シャッフルされたコードブロックを正しい順序に並べましょう。
   - **タップ** → 選択肢エリア↔答えエリアを即座に移動
   - **長押し → ドラッグ** → 細かい順序を調整

3. **自動判定 & スコア**  
   全ブロックが答えエリアに揃うと自動で正解判定。ヒントを使わず解けると **S ランク**！

4. **連続チャレンジで streak を伸ばそう**  
   毎日解いてストリーク（🔥 日連続）を記録しましょう。

## ✨ 主な機能

| 機能 | 説明 |
|------|------|
| 📱 タップ移動 | ブロックをタップするだけで上下エリア間を瞬時に移動 |
| 👆 長押しドラッグ | 長押しでドラッグ＆ドロップに切り替え、細かく順序を変更 |
| 📌 ピン留めブロック | `#include` などの固定行は自動配置済みで移動不可 |
| 💡 ヒント機能 | 最大 3 つのヒントで詰まっても安心（ヒント数でスコアに影響） |
| 🏳 諦めるボタン | 詰めてしまったときにボタン一つで正解を表示（ホームに戻る） |
| ✅ 自動正解判定 | 全ブロック配置完了時に即時判定 |
| 📖 解説モーダル | 正解後に時間計算量・空間計算量・ポイント解説を表示 |
| 🔥 ストリーク | 毎日クリアすることで連続日数を記録 |
| 🏆 スコアリング | S / A / B / C ランクで達成度を評価 |
| 📱 スマホ最適化レイアウト | 問題文・コード欄を別ページに分け、スワイプで切り替え可能 |
| 🖥️ PC版タブUI | デスクトップでも「問題」「コード」タブで快適に切り替え可能 |

## 🧩 問題の追加方法

1. **問題ファイルを作成**  
   `js/data/<カテゴリ>/<問題ID>.js` を作成し、以下の形式で記述します。

```javascript
(window.PROBLEMS_REGISTRY = window.PROBLEMS_REGISTRY || []).push({
  id: 'basic_005',          // 一意のID
  title: '問題タイトル',
  category: 'basic',        // カテゴリID
  difficulty: 1,            // 1〜5
  language: 'cpp',          // 'cpp' | 'python' | 'java' など
  description: '問題の説明文',
  pinnedCode: ['#include <bits/stdc++.h>', 'using namespace std;'],  // 固定行（省略可）
  blocks: [
    { id: 0, code: 'void example() {' },
    { id: 1, code: '    // 処理' },
    { id: 2, code: '}' },
  ],
  // 部分順序制約（[a, b] = 「ブロックa はブロックb より前」）
  partialOrder: [
    [0, 1], [1, 2],
  ],
  hints: [
    'ヒント1',
    'ヒント2',
    'ヒント3',
  ],
  explanation: {
    summary: '解説の概要',
    points: ['ポイント1', 'ポイント2'],
    complexity: { time: 'O(N)', space: 'O(1)' },
    tip: '実践的なアドバイス',
  },
});
```

2. **`index.html` に script タグを追加**（`registry.js` の読み込みより前に追加）

```html
<script src="js/data/basic/basic_005.js"></script>
```

3. **新しいカテゴリが必要な場合** は `js/data/registry.js` の `CATEGORIES` 配列に追記します。

---

## 🔧 正解判定について

- **`partialOrder` 方式**（推奨）: `[a, b]` の組みで「ブロック a はブロック b より前」という制約を指定します。`}` など見た目が同じブロックが複数ある問題に対応しており、論理的に等価な複数の答えを自動で正解と判定します。
- **`correctOrders` 方式**（従来）: 正解の並び順を配列で直接指定します。
